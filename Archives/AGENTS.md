# Archives

## Purpose

Archives store finished, abandoned, or superseded content from the other three PARA categories.

Moving items here keeps Areas, Projects, and Resources clean and focused.

## What belongs here

- Completed projects with no further action.
- Areas of responsibility you no longer maintain.
- Superseded reference materials.
- Old versions of notes that have been replaced.

## What does not belong

- Anything still active or relevant. Keep in **Projects**, **Areas**, or **Resources**.
- Temporary drafts. Delete instead.

## Subdirectory structure

Archives mirrors the active PARA structure:

- `Archives/Projects/` — completed or abandoned projects.
- `Archives/Areas/` — outdated areas of responsibility.
- `Archives/Resources/` — superseded reference material.

## Agent usage

- Move notes to Archives only when user confirms they are inactive.
- Do not delete notes without user approval.
- Archived notes keep their original frontmatter. Add `archived: true` if helpful.

## File convention

Same as the source directory: `kebab-case-title.md` with YAML frontmatter.
