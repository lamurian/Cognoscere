Several lightweight open-source browsers offer native vim-like keybindings for keyboard-driven browsing. Three categories exist: standalone GUI browsers with built-in vim bindings, text-based terminal browsers, and extensions that add vim bindings to mainstream browsers.

Standalone GUI browsers include Vimb (C/WebKitGTK, minimal memory, Linux-only), qutebrowser (Python/Qt, cross-platform, most feature-rich, 11.6k stars), Vieb (Electron/JS), Luakit (Lua-scriptable WebKit/GTK+), wyeb (C/webkit2gtk), Nyxt (Lisp, Vim and Emacs bindings), and surf (~30KB binary, no tabs, MIT from suckless.org).

Terminal browsers include Lynx (oldest maintained, `-vikeys` flag), w3m (vim-like keys, also a pager), ELinks (SpiderMonkey JS support), and browsh (headless Firefox to terminal). These use ~5MB but cannot render modern JS-heavy pages.

Extensions like Vimium (Chrome), Tridactyl (Firefox), Surfingkeys (cross-browser), and Vimari (Safari) add vim bindings without switching browsers.

Best minimal-resource GUI choice is Vimb (~50-80MB per tab). Best cross-platform with features is qutebrowser (~200-400MB). Lowest resources is Lynx (~5MB memory). Easiest setup is Vimium (Chrome) or Tridactyl (Firefox). Most minimalist by philosophy is surf (~30KB, no tabs).
