Software engineering is the second most heavily impacted industry. Singh et al. (2025) directly demonstrated that code-generating LLMs exhibit Dunning-Kruger-like overconfidence, especially in low-resource programming languages, which could mislead developers who trust the outputs [@singh2025].

The rise of **vibe coding** — a paradigm where developers write code by prompting LLMs rather than writing code directly — compounds this. Sarkar and Drosos (2025) conducted the first empirical study of vibe coding and found that it does not eliminate the need for programming expertise but rather redistributes it toward context management and rapid code evaluation. Critically, they found that trust in AI tools is dynamic and contextual, developed through iterative verification rather than blanket acceptance. However, their concept of **"material disengagement"** — where practitioners orchestrate code production through AI while maintaining selective oversight — points to a risk surface where less experienced developers may lack the metacognitive calibration to know when to verify [@sarkar2025].

The Dunning-Kruger dynamic in software engineering is acute: novice programmers who generate working code through LLMs may overestimate their own programming competence, having never developed the debugging and architectural reasoning skills that expert developers possess. Koch's metacognitive decoupling model — where output improves but understanding degrades — directly applies [@koch2026].
